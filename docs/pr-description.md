# feat(map-sync): publish native client tile snapshots

## 修改目标

让客户端 Xaero 原生生成结果成为共享地图主数据源，减少服务端重复渲染和棋盘状空洞，同时保留自然加载区块的安全回退。

## 修改内容

- 从 Xaero World Map 1.25.1 的原生 MapTile 提取完整 v5 地表快照。
- 客户端限时扫描附近 ready tile；相同内容哈希去重，哈希变化立即作为新快照上传。
- 服务端按握手、包大小、速率、字节、维度、距离和加载状态校验上传，并以当前自然加载区块重算完整 v5 地表哈希，拒绝伪造视觉字段。
- 客户端快照先写 staging 文件；dirty generation 与原子替换在同一临界区提交，索引命中但 body 缺失时也会自动修复。
- 同坐标不同哈希按有序 writer 的到达顺序采用最新快照；客户端 revision 不参与上传冲突判断。
- 新探索区块先等待客户端快照，超时、校验失败或背压时才由服务端对自然加载区块回退渲染。
- 服务端渲染上限放宽，并由 MSPT 余量、2ms 安全余量、40ms 事故预算和 writer 容量自适应背压。
- dirty 任务按 64 项分页领取，避免一次领取大量任务后又逐项退回。
- 协议升级到 6，地图格式保持 5，版本升级到 `2.1.0-beta.12`。

## 不包含的内容

- 不主动或弱制加载未加载的 Minecraft 区块。
- 不使用 GPU 读取或校验世界语义数据。
- 不修改 Xaero 原生地图文件格式、路径点权限模型或旧 v5 缓存目录。

## 风险等级

High

## 行为变化

修改前：客户端只发送本地完成提示，服务端重新扫描世界生成共享瓦片，吞吐受固定 16 块和 25ms 限制。

修改后：客户端上传 Xaero 原生不可变快照，服务端校验、整合和主动分发；服务端渲染只作为延迟回退，并按实时负载动态吞吐。

## 验证方式

- [x] 单元测试：192/192 通过
- [x] 集成构建
- [x] Xaero 1.25.1 反射签名测试
- [x] 哈希更新、重复去重、dirty generation 和 writer 背压回归测试
- [x] 畸形包、包数和压缩字节预算测试
- [ ] 双客户端真实高速飞行、换维度、退出重进视觉验证
- [ ] 多玩家、多维度持续压力测试

## 评测结果

基线结果：公共瓦片主要由服务端生成，固定 16 块/tick，客户端高速跑图时容易形成等待空洞。

修改后结果：192 项自动化测试和完整构建通过；客户端可上传完整 Xaero 快照，服务端仅在客户端结果不可用时回退。真实双客户端视觉与压力结果尚待本地运行验证。

差异：地图上传增加客户端到服务端流量；服务端仍对自然加载区块采样一次用于完整性验证，但不采用不同的服务端 payload 覆盖合法客户端快照。相同哈希不会重复落盘和广播。

## 性能和成本

响应时间：新快照通过服务端 writer 落盘后主动推送；回退渲染由 MSPT 和 writer 容量动态控制。

Token 消耗：不涉及模型调用。

工具调用次数：不涉及外部 Tool 或 API。

## 安全影响

是否增加权限：否。

是否增加数据访问：客户端读取本机 Xaero 已生成的地表 tile；不上传实体、容器、NBT 或世界存档。

是否增加网络访问：增加协议内 C2S tile 上传，不增加外部网络访问。

是否涉及敏感信息：否。

## 发布方式

使用 `2.1.0-beta.12` 灰度发布。服务端和所有客户端必须同步升级协议 6，并监控 MSPT、上传拒绝率、writer 队列、网络流量和 fallback 比例。

## 回滚方式

停止服务端，将服务端和客户端 JAR 同步回滚到 `2.1.0-beta.11`。地图格式仍为 v5，现有 `tiles-v5`、`map_tile_index-v5.json` 和客户端 v5 缓存可保留；回滚前备份 `<world>/xaero-mapsync_r/`。
