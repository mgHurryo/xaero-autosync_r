# fix(client): manage sharing in Xaero waypoint screen

## 修改目标

修复共享路径点颜色写入导致 Xaero 双客户端崩溃的问题，并按实际交互要求移除独立路径点创建/导入界面，将共享管理直接接入 Xaero 原生路径点管理页。

## 修改内容

- 将旧 RGB 颜色迁移为 Xaero 22.11.1 的 `0..15` 调色板索引；`0x55FFFF` 精确迁移为索引 11。
- 在服务端加载、创建和客户端落地三处防御非法颜色。
- 在 Xaero 原生 `GuiWaypoints` 页面加入可见性状态、公开共享、条件队伍共享和删除共享按钮。
- 点击时读取 Xaero 当前唯一选中的原生路径点、显示世界、集合、坐标、符号和颜色。
- 创建者客户端以隐藏 UUID 绑定并复用原生路径点；其他客户端维护服务器同步副本。
- 已共享路径点显示锁图标，共享和 Xaero 修改按钮变灰；服务器拒绝所有更新，但允许任意可见玩家确认后删除。
- 客户端会把被本地编辑的远端共享点恢复为服务器版本，且不创建重复副本。
- 移除 `]` 快捷键、独立管理页、手工新建页和本地导入页。
- TEAM 路径点加入 Xaero 客户端 reconcile。
- 更新回归测试、本地联调步骤、README 和预发布版本 `2.1.0-beta.3`。

## 不包含的内容

- 不升级 Minecraft、Fabric 或固定的 Xaero 版本。
- 不改变同步协议号或地图格式号。
- 不自动上传任何未选择的私人路径点。
- 不修改 Xaero World Map 1.25.1 的既有签名兼容问题。

## 风险等级

Medium

## 行为变化

修改前：独立界面可以手工创建并导入路径点；RGB 被当作 Xaero 颜色索引，打开路径点页时可能数组越界崩溃。

修改后：只能在 Xaero 原生管理页中选择现有路径点后共享；所有落地颜色均为合法调色板索引；共享后显示锁定状态且不可编辑，只能删除。

## 验证方式

- [x] 单元测试：105/105 通过
- [x] 集成构建：`gradlew.bat clean build`
- [x] 输入、权限、revision 和颜色迁移回归测试
- [ ] 双客户端 GUI 与共享生命周期人工端到端验收
- [ ] 大型 TNT/活塞/光照压力测试

## 评测结果

基线结果：两份客户端崩溃报告均为 `COLORS[5636095]` 数组越界。

修改后结果：构建和 105 个测试通过；颜色迁移、TEAM reconcile、创建者原生对象复用、持久身份、锁定、删除及失败重试均有自动化覆盖。

差异：真实 Xaero 页面布局和双客户端网络生命周期仍需本地三进程人工验收。

## 性能和成本

响应时间：共享点击只读取一个原生路径点并发送一条已有网络消息；未增加后台扫描频率。

Token 消耗：不涉及模型调用。

工具调用次数：每次明确点击产生一次创建或删除请求。

## 安全影响

是否增加权限：否。

是否增加数据访问：只在点击时读取 Xaero 当前选中的一个路径点。

是否增加网络访问：否，复用现有 Fabric 网络通道。

是否涉及敏感信息：不涉及密钥；路径点坐标仍按玩家明确操作提交。

## 发布方式

预发布版本，在本地双客户端环境验收后灰度部署。

## 回滚方式

停止服务端，回退本提交和 Mod JAR，并恢复对应的 `<world>/xaero-mapsync_r/` 数据备份。旧版本重新启用前必须确认服务端和客户端 Xaero 文件中不存在大于 15 的受管路径点颜色。
