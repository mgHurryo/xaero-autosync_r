# feat(xaero): support all Fabric map releases for Minecraft 1.17.1

## 修改目标

将原先仅支持 Xaero World Map 1.25.1 与 Xaero Minimap 22.11.1 的适配器扩展到 Minecraft 1.17.1 下全部已发布 Fabric 版本，并保持未知反射合约在写入前失败关闭。

## 修改内容

- World Map 支持 `1.14.5.2` 至 `1.37.8` 的 74 个 Fabric 版本，按九类二进制签名自适应反射参数。
- Minimap 支持 `21.12.5.1` 至 `23.9.7` 的 80 个 Fabric 版本，为早期版本增加安全的单例回退。
- 增加完整发布版本门禁测试、代表签名族 ASM 合约测试及带 SHA-256 校验的 CI fixtures。
- 本地联调脚本可按文件名选择任意受支持 Fabric 版本组合，默认使用最后发布版本。
- 更新 Fabric 元数据、适配器版本标识与兼容文档。

## 不包含的内容

- 不支持 Forge、NeoForge、Quilt 专用文件或其他 Minecraft 版本。
- 不修改地图同步协议版本、v6 存储格式、权限策略、网络限流或服务端数据。
- 不执行发布、生产部署或生产数据迁移。

## 风险等级

Medium

## 行为变化

修改前：非 World Map 1.25.1 或 Minimap 22.11.1 会在版本门禁处禁用对应适配器。

修改后：Minecraft 1.17.1 已发布 Fabric 版本进入签名校验，并根据实际类结构选择参数；超出边界或签名未知时仍在任何 Xaero 数据修改前禁用适配器。

## 验证方式

- [x] 单元测试
- [x] 集成测试（字节码合约）
- [ ] 智能体评测（不适用）
- [x] 安全测试（版本边界、未知签名失败关闭、fixture SHA-256）
- [ ] 人工测试（需要逐签名族启动 Fabric 客户端）

## 评测结果

基线结果：仅两个固定版本可通过门禁。

修改后结果：74 个 World Map 与 80 个 Minimap 发布版本全部通过版本清单回归；九类 World Map 与 Minimap 首尾版本代表 JAR 通过 ASM 合约测试。

差异：新增 152 个可接受的上游版本；未改变同步数据格式。

## 性能和成本

响应时间：启动时增加常数级可选反射查找；同步热路径仅增加少量已缓存的参数分支。

Token 消耗：不适用。

工具调用次数：运行时网络和工具调用次数不变。

## 安全影响

是否增加权限：否。

是否增加数据访问：否，仍只访问现有 Xaero 地图与明确共享的路径点。

是否增加网络访问：运行时否；CI 从既有上游 CDN 下载固定哈希的 Fabric 测试 JAR。

是否涉及敏感信息：否。

## 发布方式

先在本地/开发环境按九类签名代表版本人工启动验证，再直接发布 Mod JAR；不自动部署。

## 回滚方式

停止客户端和服务端，恢复上一版 Mod JAR 或回滚本 PR 提交。此次修改没有数据库迁移或存储格式变化，无需回滚 `tiles-v6`、Xaero 地图目录或路径点数据；如人工联调发现上游私有 API 写入异常，使用联调前备份恢复对应客户端 `XaeroWorldMap` 目录。
